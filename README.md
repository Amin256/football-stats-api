Football Stats API
## Project Overview

This project implements a RESTful API for football match statistics and analytics using **FastAPI**, **SQLAlchemy**, and **SQLite**.

The API allows users to store football teams and match results and provides analytical endpoints for computing statistics such as league tables, team form, defensive performance and home vs away performance.

## Features

### Data Management
- Create and retrieve football teams
- Create and retrieve match results
- Store match data in a relational database

### Analytics Endpoints
- League table calculation (points, wins, draws, losses, goal difference)
- Top scoring teams
- Best defensive teams
- Last 5 match results (team form)
- Home vs away performance comparison

## Technology Stack

The following technologies were used to implement the project:

- **Python** – main programming language  
- **FastAPI** – web framework used to build the REST API  
- **SQLAlchemy** – ORM used for database interaction  
- **SQLite** – lightweight relational database used for data storage  
- **Uvicorn** – ASGI server used to run the API  

FastAPI was chosen because it automatically generates **OpenAPI/Swagger documentation**, allowing easy testing and clear documentation of all API endpoints.

## Setup Instructions

### 1. Clone the repository

git clone https://github.com/Amin256/football-stats-api.git
cd football-stats-api

### 2. Create a virtual environment

python -m venv venv

Activate the environment:

**Windows**

venv\Scripts\activate

**Mac/Linux**

source venv/bin/activate

### 3. Install dependencies

pip install -r requirements.txt

### 4. Run the API

Start the server using Uvicorn:

uvicorn app.main:app --reload

The API will run at:

http://127.0.0.1:8000

## API Documentation

Interactive API documentation is automatically generated by **FastAPI** using the OpenAPI specification
Once the server is running, the documentation can be accessed at:

http://127.0.0.1:8000/docs

This interface allows users to:

- View all available API endpoints  
- Inspect request parameters  
- See example responses  
- Test endpoints directly from the browser  

A static version of the API documentation is also included in this repository:

docs/api_documentation.pdf

## Example API Endpoints

### Get all teams

GET /teams
Example response:

```json
[
  {
    "id": 1,
    "name": "Arsenal",
    "league": "Premier League",
    "country": "England"
  }
]
```

### Get league table

GET /analytics/league-table

Example response:

```json
[
  {
    "team": "Arsenal",
    "played": 38,
    "wins": 28,
    "draws": 5,
    "losses": 5,
    "goals_for": 91,
    "goals_against": 29,
    "goal_difference": 62,
    "points": 89
  }
]
```

### Get team form (last 5 matches)

GET /team-form/{team_id}

Example response:

```json
{
  "team": "Arsenal",
  "form": ["W", "W", "D", "L", "W"],
  "points": 10
}
```

## Dataset

The project includes a script for importing a Premier League dataset containing match results.
To import the dataset:

python -m scripts.import_dataset

This populates the database with teams and match results used by the analytics endpoints.
